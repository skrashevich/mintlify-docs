Mock Telegram Bot API server that replaces real Telegram with LLM-generated responses. Point your bot at this server instead of api.telegram.org and get realistic AI-powered replies for testing.

<Card title="GitHub" icon="github" href="https://github.com/skrashevich/telegram-mock-ai">Source code</Card>

## Key Features

- **Drop-in Replacement** — implements 28 Bot API methods including getMe, sendMessage, getUpdates, sendPhoto, sendDocument, and more.
- **LLM-Powered Responses** — uses any OpenAI-compatible API to generate contextual replies to bot messages.
- **Proactive Mode** — simulates users who initiate conversations. Configurable styles: curious, tester, confused, spammer, polite.
- **Multi-Chat Simulation** — creates private chats, groups, and supergroups with virtual users.
- **Admin API** — REST endpoints for managing virtual users, chats, and monitoring message flow.
- **Webhook & Polling** — supports both getUpdates long polling and webhook delivery.

## Quick Start

```bash
# Docker
docker run -p 8081:8081 -e OPENAI_API_KEY=your-key ghcr.io/skrashevich/telegram-mock-ai

# From source
go install github.com/skrashevich/telegram-mock-ai@latest
telegram-mock-ai --openai-key your-key
```

Point your bot to `http://localhost:8081/bot<TOKEN>/` instead of `https://api.telegram.org/bot<TOKEN>/`.

## Configuration

| Flag | Default | Description |
|------|---------|-------------|
| `--addr` | `:8081` | Listen address |
| `--openai-key` | | OpenAI API key |
| `--openai-model` | `gpt-4` | Model to use |
| `--proactive` | `false` | Enable proactive messaging |
| `--style` | `curious` | Proactive message style |
